Key points

- Consistency model: eventual consistency for distributed merges; supports tunable replication and quorum settings.
- Data ingestion: high-throughput inserts (batch and streaming), supports bulk loading, Kafka integration, and HTTP/Native clients.
- SQL dialect: ANSI SQL-like with many analytical extensions (array/map types, window functions, approximate functions like uniqExact/uniq).
- Indexing: sparse primary key (data skipping indices) and secondary "data skipping" indexes; no traditional B-tree indexes.
- Use cases: time-series analytics, clickstream and event analytics, BI dashboards, ETL aggregation, real-time monitoring.
- Integrations: connectors for Kafka, RabbitMQ, Spark, Airflow, Grafana, Prometheus exporters, JDBC/ODBC drivers.
- Deployment: runs on Linux; available as self-hosted, Kubernetes operators, and managed services (various cloud providers offer hosted ClickHouse).
- Licensing: open source under the Apache 2.0 license (core); some commercial distributions/extensions exist.

When to choose ClickHouse
- You need extremely fast OLAP queries over large append-heavy datasets.
- Workloads are read/aggregate heavy (GROUP BY, ORDER BY, top-N, time-windowed analytics).
- Low-latency dashboarding or real-time analytics is required.
